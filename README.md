# 無料券管理アプリ（スタンドアロン版）

Claudeのアーティファクト上では `<input type="file">` によるネイティブの写真アルバム／カメラ呼び出しが
サンドボックスの制約でブロックされていたため、通常のWebアプリ（PWA）として動かせるように
プロジェクト一式を用意しました。ここではアーティファクトの制約がないので、写真選択は正常に動作します。

## 含まれているもの

- Vite + React によるシンプルな構成
- `vite-plugin-pwa` によるPWA対応（ホーム画面に追加してアプリのように使えます）
- クーポンの登録・一覧・詳細・使用済み管理（元の要件定義のMVP範囲）
- データはブラウザの `localStorage` に保存されます（端末内保存、端末をまたいだ同期はありません）

## ローカルで試す（Claude Codeでの手順の目安）

```bash
npm install
npm run dev
```

表示されるURL（例: `http://localhost:5173`）にスマホからアクセスすれば、写真アルバムへのアクセスを含めて
動作確認できます。同じWi-Fiに繋がっていれば `--host` 経由でスマホから直接開けます（`server.host: true` を
`vite.config.js` に設定済みです）。

## 本番公開・スマホのホーム画面に追加する

実際にホーム画面から使うアプリとして使いたい場合は、どこかにデプロイしてHTTPS化する必要があります
（PWAのインストールやカメラ・写真アクセスはHTTPS必須です）。

```bash
npm run build
```

`dist/` フォルダができるので、Vercel・Netlify・Cloudflare Pagesなど任意の静的ホスティングにアップロードして
ください。公開後、スマホのSafari／Chromeで開き、「ホーム画面に追加」からインストールすると、
ネイティブアプリのように使えます。

## 今後のカスタマイズ候補

- `public/icons/` のアイコンは仮のものです。ブランドに合わせて差し替えてください。
- OCRによる商品名・期限の自動読み取り（要件定義の「将来追加」項目）
- 期限が近づいたときのプッシュ通知（Web Push、要HTTPS + Service Worker追加実装）
- 端末をまたいだデータ同期をしたい場合は、`localStorage` の代わりにFirebaseなどのバックエンドが必要になります
